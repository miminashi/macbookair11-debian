# MacBook Air 2015 (Debian 13) WiFi接続問題 — 調査・修正レポート

**日付**: 2026-04-01
**対象ホスト**: macbookair2015.lan (miminashi@)
**OS**: Debian GNU/Linux 13 (trixie), kernel 6.12.74+deb13+1-amd64

---

## 1. 症状

- WiFi（SSID: OpenWrt）に接続できない
- NetworkManagerが接続を繰り返し試みるが、毎回タイムアウトで失敗
- エラーメッセージ: `ssid-not-found`

## 2. ハードウェア・ドライバ構成

| 項目 | 値 |
|---|---|
| WiFiチップ | Broadcom BCM4360 802.11ac Dual Band (rev 03) |
| PCI | 03:00.0 |
| ドライバ | `wl` (broadcom-sta-dkms 6.30.223.271-26) |
| インターフェース | `wlp3s0` (MACアドレス: 98:e0:d9:8d:20:5d) |
| wpa_supplicant | wpasupplicant 2:2.10-24 |
| NetworkManager | 1.52.1 |

## 3. 原因分析

### 3.1 直接原因

`wl`ドライバ（Broadcomプロプライエタリドライバ）が `WPA-PSK-SHA256` キー管理方式をサポートしていない。

### 3.2 発生メカニズム

1. **OpenWrt APの設定**: WPA2 + WPA3 トランジションモード（`WPA2 WPA3`）でブロードキャスト
2. **NetworkManagerの動作**: APのセキュリティ能力を検出し、wpa_supplicantの設定に `key_mgmt=WPA-PSK WPA-PSK-SHA256` を自動的に設定
3. **ドライバの制限**: `wl`ドライバは`WPA-PSK-SHA256`を処理できず、`ERROR @wl_set_key_mgmt`と`invalid cipher group (1027076)`エラーを発生
4. **結果**: 認証プロセスが完了せず、NetworkManagerがタイムアウト後に`ssid-not-found`として処理

### 3.3 ログ証拠

**dmesg**:
```
ERROR @wl_set_key_mgmt :
invalid cipher group (1027076)
```
（43回繰り返し）

**NetworkManager journal**:
```
Config: added 'key_mgmt' value 'WPA-PSK WPA-PSK-SHA256'
Activation: (wifi) association took too long, failing activation
state change: config -> failed (reason 'ssid-not-found')
```

### 3.4 NetworkManagerでの回避不可

以下の設定変更を試みたが、NM 1.52.1はAPの能力に基づいて`WPA-PSK-SHA256`を自動追加し続けた:

- `802-11-wireless-security.pmf=disable` → `ieee80211w=0`は追加されるが`key_mgmt`には`WPA-PSK-SHA256`が残存
- 明示的な`proto=rsn`, `pairwise=ccmp`, `group=ccmp`指定 → 効果なし

### 3.5 切り分け確認

wpa_supplicantを直接実行し、`key_mgmt=WPA-PSK`のみで`nl80211`ドライバを使用した場合、**接続成功**:
```
Trying to associate with d8:ec:5e:f8:c4:8b (SSID='OpenWrt' freq=5680 MHz)
Associated with d8:ec:5e:f8:c4:8b
WPA: Key negotiation completed with d8:ec:5e:f8:c4:8b [PTK=CCMP GTK=CCMP]
CTRL-EVENT-CONNECTED - Connection to d8:ec:5e:f8:c4:8b completed
```

## 4. 実施した修正

NetworkManagerによるWiFi管理を回避し、wpa_supplicant + systemd-networkdで直接管理する構成に変更。

### 4.1 NetworkManagerからwlp3s0を除外

**ファイル**: `/etc/NetworkManager/conf.d/unmanage-wifi.conf`
```ini
[keyfile]
unmanaged-devices=interface-name:wlp3s0
```

### 4.2 wpa_supplicant設定

**ファイル**: `/etc/wpa_supplicant/wpa_supplicant-nl80211-wlp3s0.conf`（パーミッション 600）
```
ctrl_interface=/run/wpa_supplicant
ctrl_interface_group=0
update_config=1
country=JP

network={
    ssid="OpenWrt"
    key_mgmt=WPA-PSK
    psk="meganekkomoemoe"
    proto=RSN
    pairwise=CCMP
    group=CCMP
}
```

### 4.3 systemd-networkd DHCP設定

**ファイル**: `/etc/systemd/network/25-wlp3s0.network`
```ini
[Match]
Name=wlp3s0

[Network]
DHCP=yes
IPv6AcceptRA=yes

[DHCPv4]
RouteMetric=600

[DHCPv6]
RouteMetric=600
```

RouteMetric=600により、有線接続（デフォルトメトリック）が優先される。

### 4.4 サービス有効化

```bash
# NM OpenWrtプロファイル削除
nmcli connection delete OpenWrt
systemctl restart NetworkManager

# wpa_supplicant + systemd-networkd 有効化
systemctl enable --now systemd-networkd
systemctl enable --now wpa_supplicant-nl80211@wlp3s0
```

修正後の新しい`ERROR @wl_set_key_mgmt`エラーは発生していない。

## 6. 今後の留意事項

- **WiFi SSID/パスワード変更時**: `/etc/wpa_supplicant/wpa_supplicant-nl80211-wlp3s0.conf`を直接編集し、`systemctl restart wpa_supplicant-nl80211@wlp3s0`で反映
- **別のWiFiネットワーク追加**: 同ファイルに`network={}`ブロックを追加
- **有線LAN切断後**: WiFi経由でSSH接続する場合は`ssh miminashi@192.168.1.145`を使用
- **`wl`ドライバの制限**: WPA3（SAE）専用ネットワークには接続不可。WPA2またはWPA2+WPA3トランジションモードのネットワークのみ対応
