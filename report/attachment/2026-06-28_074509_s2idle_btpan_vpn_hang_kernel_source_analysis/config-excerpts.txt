# ===== 最重要: device-suspend ウォッチドッグ無効 =====
# CONFIG_DPM_WATCHDOG is not set
  → device の ->suspend/->resume callback が無限ブロックしても殺すものが無い。
    suspend_console() が dpm_suspend_start() の前に走る (kernel/power/suspend.c:505 < :507) ため、
    ブロックは「無ログ・PM: suspend exit 欠落・永久ハング (要強制電源断)」になる = 観測と一致。
